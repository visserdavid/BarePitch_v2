- Documentation alignment pass.

Security checklist:
- All write routes require CSRF.
- All write routes check policy server-side.
- SQL uses prepared statements.
- No stack traces are visible to users.
- Sessions use secure settings.
- Magic links are one-time and expire.
- Magic-link tokens are stored hashed.
- Login and public livestream endpoints are rate-limited.
- Public token pages send no-store and noindex protections.
- Public token failures use generic messaging.
- No temporary developer login path remains enabled outside local development.
- Production HTTPS deployments send HSTS.
- Session idle and absolute lifetime rules are implemented.
- Invalid access attempts fail safely.

Data consistency checklist:
- Score is source-derived from events.
- Shootout score is separate from normal score.
- Lineup current state remains valid after substitutions and red cards.
- Playing time is stored in seconds and remains consistent.
- Finished-match corrections recalculate affected derived data.
- Locking prevents silent overwrite.
- Audit logging exists for every finished-match correction.

MVP scope discipline:
- Do not build advanced analytics.
- Do not build ratings unless already explicitly required by current code/docs for MVP acceptance; ratings are not required before first MVP release.
- Do not build training management unless already required for a specific MVP statistic currently exposed; training is full-scope but not MVP required.
- Do not expand dashboard beyond navigation and next actionable item support.
- Do not introduce realtime websocket infrastructure.

Acceptance criteria:
- The complete coach flow works: login, select team, manage players, create match, prepare lineup, add guests, start match, register events, substitute, handle cards, finish, share livestream, correct finished data, view basic statistics.
- Administrator can perform minimum club/season/phase/team/user/role setup.
- Authorization matrix is enforced across MVP routes.
- No MVP write route lacks CSRF.
- Public livestream endpoints use safe token behavior and headers.
- Basic statistics are correct for implemented MVP sources.
- Translation helper is used for visible MVP UI labels in touched screens.
- Tests cover the MVP acceptance scenario from the test scenarios doc plus security scenarios SE-01 through SE-04.
- Regression tests cover score integrity, lineup integrity, red-card restrictions, correction behavior, audit writes, locks, livestream expiry/stop, and unauthorized access.
