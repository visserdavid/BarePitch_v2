Goal:
Allow a coach to run regular time for a match: start, register goals, manage periods, finish, and view the result.

Implement these slices:
- Phase 6 live match core.
- Live match screen.
- Score header.
- Match timer display.
- Current lineup display.
- Live timeline.
- Goal registration modal.
- Own regular goal.
- Opponent regular goal.
- Optional assist for eligible own goals.
- Optional 3 by 3 goal zone matrix.
- End first half.
- Start second half.
- End second half.
- Finish match.

Critical behavior:
- Only prepared matches can start.
- Starting a match creates the appropriate regular-time period state.
- Score starts at 0-0 and is recalculated from event source.
- Own regular goals increase BarePitch score.
- Opponent regular goals increase opponent score.
- Assists are optional and must follow eligibility rules.
- Goal zone selection is optional but must use documented zone values.
- Period transitions must follow the critical behavior spec.
- Finished match must remain viewable and must not restart.
- Multi-entity writes must be transactional.

Authorization and security:
- Coach and administrator can perform live match writes where documented.
- Unauthorized roles must be blocked server-side.
- Every live write route requires CSRF.
- Validation errors fail safely without partial state.

UI requirements:
- Critical period and finish actions require swipe interaction and confirmation modal where specified.
- Live action controls are contextual in-screen controls, not permanent top-level navigation.
- Repeated live controls may use icon-first design when clarity is preserved.
- Primary bottom navigation labels remain visible.

Explicitly do not implement yet:
- Substitutions.
- Cards.
- Penalties during match.
- Extra time.
- Penalty shootout.
- Livestream.
- Corrections.
- Ratings.

Acceptance criteria:
- Coach can start a prepared match.
- Starting a planned match is rejected.
- First half starts immediately or according to documented start behavior.
- Coach can register own and opponent goals.
- Score updates from persisted events and remains correct after refresh.
- Goal appears in timeline.
- Optional assist and goal zone persist when provided.
- Coach can end first half, start second half, end second half, and finish match.
- Finished match summary is viewable.
- Tests cover MS-01 through MS-05, SC-01, SC-02, score recalculation, invalid period transitions, unauthorized live writes, and CSRF rejection.

Before finishing:
- Run available tests and PHP syntax checks.
- Verify score is never maintained only as a blind increment.
- Keep this milestone limited to regular-time live match core.
