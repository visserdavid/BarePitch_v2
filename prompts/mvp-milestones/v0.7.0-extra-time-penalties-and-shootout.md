Goal:
Support match flows beyond regular time while keeping match score, penalty events, and shootout data consistent.

Implement these slices:
- Phase 9 penalties during match.
- Phase 10 extra time and penalty shootout.
- Scored penalty during regular/extra time.
- Missed penalty during regular/extra time.
- Penalty taker selection.
- Optional zone for scored penalties.
- Extra-time start action.
- Extra-time period ending.
- Penalty shootout start action.
- Shootout attempt storage.
- Attempt order and round number.
- Scored/missed shootout outcomes.
- Sudden death marker/support.
- Automatic ending detection.
- Manual ending confirmation.

Penalty during match rules:
- Scored penalty updates match score.
- Missed penalty does not update match score.
- No assist option is available for penalties.
- Penalty events appear in timeline.
- Optional zone is allowed for scored penalties when documented.

Extra time rules:
- Coach can choose extra time after regular time where documented.
- Extra time has two periods.
- Extra-time duration comes from match configuration.
- Manual ending is required.

Shootout rules:
- Coach can choose penalties after regular time or after extra time where documented.
- Shootout attempts are stored separately from match events.
- Shootout score does not affect normal match score.
- Sent-off players cannot take penalties.
- Duplicate attempt order is rejected.
- Automatic ending requires confirmation.
- Manual ending requires confirmation.

Authorization and security:
- All writes require authenticated, authorized actors.
- CSRF required for all state-changing routes.
- Multi-entity state transitions must be transactional.
- Public exposure is not part of this milestone.

UI requirements:
- Extra-time and shootout transitions are critical actions and need confirmation.
- Shootout UI should make round, taker, team, and outcome clear.
- Do not overload the regular live match controls with permanent shootout controls before they are relevant.

- Ratings.
