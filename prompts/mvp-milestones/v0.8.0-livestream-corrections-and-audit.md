Goal:
Provide public match viewing without login and allow authorized post-match corrections with audit logging.

Implement these slices:
- Phase 11 livestream.
- Phase 12 finished match corrections and audit.
- Match edit locking if not already implemented enough to prevent correction conflicts.
- Livestream token creation when match becomes active.
- Public livestream route.
- Public livestream data polling route.
- Score, timeline, and phase display.
- Livestream expiration.
- Manual livestream stop.
- Optional token rotation if specified by route docs.
- Finished match correction screen.
- Event correction: scorer, assist, zone, penalty outcome.
- Substitution correction.
- Shootout attempt correction.
- Score recalculation after correction.
- Audit logging for every finished-match correction.

Livestream rules:
- Public link works without login.
- Token must not be stored in plaintext.
- Starts when match becomes active.
- Default expiration is 24 hours after finish.
- Maximum expiration is 72 hours.
- Coach can stop livestream manually.
- Expired or stopped links deny access with generic failure behavior.
- Corrections are visible while livestream is active.
- Public pages send no-store/noindex protections.
- Public token endpoints must be rate-limited or prepared for rate limiting according to security docs.

Correction rules:
- Only coach and administrator can correct finished matches.
- Trainer and team manager cannot correct finished match data.
- Match remains finished after correction.
- Corrections must recalculate affected derived/cached data.
- Corrections must be audited with user, timestamp, changed values, and target entity.
- No partial correction without recalculation.

Locking rules:
- Only one active editor per match.
- Recommended timeout is 2 minutes.
- Lock refresh must work.
- No silent overwrite is allowed.

UI requirements:
- Public livestream is readable on mobile and refreshes by polling.
- Correction UI must make the edited item and old/new values clear.
- Critical correction saves should be deliberate and recover validation errors safely.

Explicitly do not implement yet:
- Ratings.
- Training.
- Advanced statistics.
- Extended dashboard.
- Full multi-language coverage beyond existing translation foundation.

Acceptance criteria:
- Public livestream link works without login.
- Livestream shows current score, phase, and timeline.
- Polling updates the livestream after a new event.
- Expired livestream link is inaccessible.
- Manually stopped livestream link is inaccessible.
- Corrections appear in active livestream.
- Coach and administrator can correct finished match events, substitutions, and shootout attempts.
- Trainer and team manager cannot correct.
- Score recalculates after correction.
- Each correction writes audit log entry.
- Match remains finished after correction.
- Lock acquisition, second-user blocking, expiry replacement, and refresh work.
- Tests cover LS-01 through LS-04, AU-01 through AU-03, LK-01 through LK-04, SC-06, unauthorized correction blocking, public token generic failure, and no-store/noindex headers.

Before finishing:
- Run available tests and PHP syntax checks.
- Verify token storage is hashed and public failure behavior does not reveal unnecessary details.
- Keep non-MVP analytics, ratings, and training out of this milestone.
